## 📌 Project Overview

This project analyzes the GitHub development history of the **Scala programming language** using pull request data. The goal is to explore contribution trends, project activity, important files, and developer involvement over time.

The project uses real GitHub repository data and applies Python-based data analysis techniques to understand how the Scala language evolved through open-source collaboration.

## 🎯 Objective

The main objective of this project is to investigate Scala's GitHub history and answer questions such as:

- How active was the Scala project over time?
- Which files were changed most frequently?
- Who were the most active contributors?
- How did contribution activity change across different years?
- What patterns can be identified from pull request data?

## 📊 Dataset

The project uses multiple CSV files containing Scala GitHub pull request data.

| Dataset | Description |
|---|---|
| `pulls.csv` | Main pull request dataset |
| `pulls_2011-2013.csv` | Pull request data from 2011 to 2013 |
| `pulls_2014-2018.csv` | Pull request data from 2014 to 2018 |
| `pull_files.csv` | Files changed in pull requests |

## 🧹 Project Workflow

- **Data Loading:** Imported multiple CSV datasets into Python.
- **Data Cleaning:** Combined, cleaned, and prepared pull request data for analysis.
- **Exploratory Data Analysis:** Explored contribution trends, changed files, and developer activity.
- **Visualization:** Created charts to represent GitHub activity and contribution patterns.
- **Insights:** Identified important trends in the Scala open-source development history.

## 🔍 Key Findings

- Scala has a rich GitHub contribution history with activity spread across multiple years.
- Pull request data helps reveal contributor activity and development trends.
- Some files were changed more frequently, showing important areas of the codebase.
- Open-source project history can be better understood through data analysis and visualization.
